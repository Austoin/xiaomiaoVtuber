# nanobot 项目结构与操作说明

本文用于快速理解本仓库的主要目录、关键文件和常见操作入口。范围采用“顶层目录 + Python 主包一级模块 + WebUI/Bridge + 常见操作步骤”，不逐个展开所有源码文件。

## 1. 项目概览

`nanobot` 是一个轻量级 AI Agent 框架，核心流程是：外部通道接收消息，消息总线转发给 Agent，Agent 调用模型 Provider，按需执行工具，并把结果返回到对应通道。

主要组成：

- `agent`：Agent 核心循环、上下文、工具调用、记忆和子 Agent。
- `channels`：Telegram、Discord、Slack、Feishu、WebSocket 等聊天平台接入层。
- `providers`：Anthropic、OpenAI 兼容接口、Azure、GitHub Copilot 等模型接入层。
- `tools`：文件、Shell、Web、MCP、Cron、Notebook、子 Agent 等工具能力。
- `session` / `memory`：会话历史、上下文压缩和长期记忆。
- `webui`：浏览器前端，用于通过 Gateway 与 nanobot 交互。
- `bridge`：WhatsApp 等外部协议桥接组件。

```text
nanobot/

```

### 2.1 目录

- `.agent/`：项目级架构、安全边界和常见问题说明目录，供开发者或 Agent 理解项目约束。
- `.nanobot/`：本地运行配置或工作区状态目录。正式用户配置通常位于 `~/.nanobot/config.json`。
- `bridge/`：TypeScript bridge 子项目，目前用于 WhatsApp bridge，构建后会随 Python 包一起分发。
- `case/`：README 中使用的演示 GIF，例如搜索、代码、日程和记忆能力展示。
- `docs/`：仓库文档目录，包含快速开始、配置、聊天应用、部署、API、SDK、记忆等说明。
- `images/`：项目图片资源，例如 README 封面、架构图、Logo、WebUI 预览图。
- `nanobot/`：Python 主包源码目录，是运行时、CLI、Provider、Channel、Tool 等核心代码所在地。
- `tests/`：自动化测试目录，覆盖 API、OpenAI 兼容接口、文档解析、SDK、Docker、工具上下文等行为。
- `webui/`：React + TypeScript 前端项目，通过 WebSocket multiplex 协议连接 Gateway。

### 2.2 顶层文件

- `README.md`：项目总入口，包含项目定位、安装、Quick Start、WebUI、架构和文档入口。
- `CLAUDE.md`：给 Claude Code 的项目工作说明，包含架构概览、开发命令和常见文件位置。
- `CONTRIBUTING.md`：贡献指南和分支策略说明。
- `SECURITY.md`：安全说明与安全边界相关信息。
- `THIRD_PARTY_NOTICES.md`：第三方依赖或组件声明，打包时作为 license 文件之一。
- `pyproject.toml`：Python 项目主配置，定义包名、版本、依赖、CLI 入口、打包规则、ruff 和 pytest 配置。
- `Dockerfile`：容器镜像构建脚本，安装运行依赖、构建 bridge，并设置容器入口。
- `docker-compose.yml`：容器编排配置，定义 `nanobot-gateway`、`nanobot-api` 和 `nanobot-cli` 服务。
- `entrypoint.sh`：容器启动入口，先检查 `~/.nanobot` 可写性，再执行 `nanobot` 命令。
- `core_agent_lines.sh`：维护辅助脚本，用于统计核心运行时代码和部分扩展目录的行数。

## 3. Python 主包 `nanobot/` 结构说明

这里的 `nanobot/` 指顶层源码包 `nanobot/nanobot/`。

### 3.1 核心运行层

- `agent/`：Agent 核心层。包含 `AgentLoop`、`AgentRunner`、上下文构建、生命周期 Hook、记忆、技能加载和子 Agent 等逻辑。
- `agent/tools/`：Agent 可调用工具集合。包括文件操作、Shell、Web 搜索/抓取、MCP、Cron、Notebook、子 Agent 等工具。
- `bus/`：异步消息总线，负责解耦外部 Channel 和 Agent 核心处理流程。
- `session/`：会话管理层，负责会话历史、上下文压缩和会话隔离。
- `cron/`：定时任务和自然语言提醒相关能力。
- `heartbeat/`：运行心跳与保活相关逻辑。

### 3.2 接入与扩展层

- `channels/`：聊天平台通道实现。不同平台把外部消息转换为统一入站消息，再把出站消息发回对应平台。
- `providers/`：模型 Provider 实现。负责对接不同 LLM 服务，并提供统一调用接口。
- `api/`：本地 API 服务层，提供 OpenAI 兼容 API 等服务入口。
- `cli/`：命令行入口实现。`pyproject.toml` 中的 `nanobot` 命令指向这里。
- `command/`：聊天内命令或内部命令分发逻辑。
- `config/`：配置 Schema 与加载逻辑，默认读取 `~/.nanobot/config.json`。
- `security/`：安全边界、网络访问和权限控制相关逻辑。

### 3.3 资源与辅助层

- `skills/`：内置技能目录。每个技能通常是一个目录，内部包含 `SKILL.md`。
- `templates/`：提示词、Agent 指令或工作区模板资源，打包时会随 Python 包分发。
- `utils/`：通用辅助函数和共享工具代码。
- `__init__.py`：Python 包标识文件。
- `__main__.py`：支持 `python -m nanobot` 的模块入口，内部调用 CLI app。
- `nanobot.py`：Python SDK façade，提供 `Nanobot.from_config()` 和 `run()` 等编程接口。

## 4. 文档目录 `docs/` 说明

`docs/README.md` 是仓库文档索引。常用文档包括：

- `quick-start.md`：安装、初始化和首次运行配置。
- `configuration.md`：Provider、工具、通道、MCP 和运行时设置。
- `chat-apps.md`：连接 Telegram、Discord、微信等聊天平台。
- `cli-reference.md`：CLI 命令参考。
- `chat-commands.md`：聊天内 Slash 命令和周期任务行为。
- `openai-api.md`：OpenAI 兼容 API 的端点、请求格式和文件上传。
- `deployment.md`：Docker、Linux service 和 macOS LaunchAgent 部署。
- `python-sdk.md`：在 Python 中以编程方式使用 nanobot。
- `memory.md`：记忆的存储、整理和恢复机制。
- `websocket.md`：WebSocket 通道和实时访问协议。
- `channel-plugin-guide.md`：自定义聊天通道插件开发指南。
- `my-tool.md`：运行时自定义工具说明。

## 5. WebUI 与 Bridge

### 5.1 `webui/`

`webui/` 是浏览器前端，技术栈是 Vite、React 18、TypeScript、Tailwind 和 shadcn/ui。开发时它会代理 `/api`、`/webui`、`/auth` 和 WebSocket 流量到 Gateway。

关键点：

- 源码目录：`webui/`
- 开发服务默认地址：`http://127.0.0.1:5173`
- Gateway 默认代理目标：`http://127.0.0.1:8765`
- 生产构建输出：`nanobot/web/dist/`
- 构建产物会由 `nanobot gateway` 服务，并可被打包进 Python wheel。

### 5.2 `bridge/`

`bridge/` 是 TypeScript 子项目，当前 package 名称为 `nanobot-whatsapp-bridge`，用于 WhatsApp bridge。它使用 Node.js 运行，构建入口是 `tsc`，运行入口是 `dist/index.js`。

常用脚本：

```bash
cd bridge
npm install
npm run build
npm run start
```

## 6. 常见操作步骤

以下命令默认在仓库根目录 `nanobot/` 下执行。

### 6.1 本地安装

```bash
pip install -e .
```

如果只想安装发布版本，可使用：

```bash
pip install nanobot-ai
```

或：

```bash
uv tool install nanobot-ai
```

### 6.2 初始化配置

```bash
nanobot onboard
```

配置文件默认位于：

```text
~/.nanobot/config.json
```

### 6.3 配置模型 Provider

示例：配置 OpenRouter API Key 和默认模型。

```json
{
  "providers": {
    "openrouter": {
      "apiKey": "sk-or-v1-xxx"
    }
  },
  "agents": {
    "defaults": {
      "provider": "openrouter",
      "model": "anthropic/claude-opus-4-6"
    }
  }
}
```

如果已有配置文件，应合并这些字段，不要直接覆盖整个文件。

### 6.4 启动 CLI Agent

```bash
nanobot agent
```

也可以使用模块方式启动：

```bash
python -m nanobot
```

### 6.5 启动 Gateway

```bash
nanobot gateway
```

Gateway 是 WebUI 和 WebSocket 通道交互的主要入口。

### 6.6 启动 WebUI 开发服务

先在 `~/.nanobot/config.json` 启用 WebSocket 通道：

```json
{
  "channels": {
    "websocket": {
      "enabled": true
    }
  }
}
```

然后分别启动 Gateway 和 WebUI：

```bash
nanobot gateway
```

```bash
cd webui
bun install
bun run dev
```

默认打开：

```text
http://127.0.0.1:5173
```

如果 Gateway 不在默认端口，可指定代理目标：

```bash
NANOBOT_API_URL=http://127.0.0.1:9000 bun run dev
```

### 6.7 构建 WebUI

```bash
cd webui
bun run build
```

构建结果会写入：

```text
nanobot/web/dist/
```

### 6.8 Docker Compose 运行

启动 Gateway 服务：

```bash
docker compose up nanobot-gateway
```

启动 OpenAI 兼容 API 服务：

```bash
docker compose up nanobot-api
```

运行 CLI profile：

```bash
docker compose --profile cli run nanobot-cli
```

`docker-compose.yml` 会把宿主机的 `~/.nanobot` 挂载到容器内 `/home/nanobot/.nanobot`。

### 6.9 运行测试与静态检查

运行全部测试：

```bash
pytest
```

运行单个测试示例：

```bash
pytest tests/test_openai_api.py::test_function -v
```

运行 ruff 检查：

```bash
ruff check nanobot/
```

WebUI 测试：

```bash
cd webui
bun run test
```

## 7. 推荐阅读路线

### 7.1 首次使用

1. 先读 `README.md`，了解项目定位和 Quick Start。
2. 再读 `docs/quick-start.md`，完成安装和初始化。
3. 根据使用场景阅读 `docs/configuration.md`、`docs/chat-apps.md` 或 `docs/deployment.md`。

### 7.2 想理解运行流程

1. 读 `CLAUDE.md` 的架构说明。
2. 看 `nanobot/bus/` 理解消息总线。
3. 看 `nanobot/agent/` 理解 Agent 处理流程。
4. 看 `nanobot/channels/` 和 `nanobot/providers/` 理解外部输入与模型输出。

### 7.3 想扩展能力

- 新增模型：从 `nanobot/providers/` 开始。
- 新增聊天平台：从 `nanobot/channels/` 和 `docs/channel-plugin-guide.md` 开始。
- 新增工具：从 `nanobot/agent/tools/` 和 `docs/my-tool.md` 开始。
- 新增技能：参考 `nanobot/skills/README.md` 和现有技能目录。

### 7.4 想修改 WebUI

1. 阅读 `webui/README.md`。
2. 启动 `nanobot gateway`。
3. 在 `webui/` 下运行 `bun install` 和 `bun run dev`。
4. 修改前端后用 `bun run test` 和 `bun run build` 验证。

## 8. 术语速查

- Agent：处理用户消息、调用模型和工具的核心执行体。
- Channel：外部聊天平台或通信入口。
- Provider：大模型服务接入层。
- Tool：Agent 可调用的能力，例如文件、Shell、Web、MCP。
- Gateway：WebUI、WebSocket 和部分服务入口依赖的运行进程。
- Bridge：连接外部协议或平台的桥接进程。
- Session：会话隔离与历史记录单位。
- Memory：长期记忆或会话上下文恢复机制。
- Skill：用 Markdown 指令扩展 Agent 能力的轻量模块。
