# 操作文档

## 1. 环境要求

- Node.js：建议使用当前 LTS 版本。
- pnpm：项目声明版本为 `pnpm@10.33.0`。
- Git：用于重新初始化仓库并上传远端。

```powershell
corepack enable
corepack prepare pnpm@10.33.0 --activate
pnpm -v
```

## 2. 安装依赖

在项目根目录执行：

```powershell
pnpm install
```

## 3. 重新建立 Git 仓库

在项目根目录执行：

```powershell
git init
git branch -M main
git add .
git commit -m "chore: initialize repository"
```

## 4. 上传到云端仓库

先在 GitHub、Gitee 或其他平台创建一个空仓库，不要勾选初始化 README。

把下面的地址替换成你的远端仓库地址：

```powershell
git remote add origin https://github.com/<你的账号>/<你的仓库>.git
git push -u origin main
```

如果使用 SSH 地址：

```powershell
git remote add origin git@github.com:<你的账号>/<你的仓库>.git
git push -u origin main
```

## 5. 常用启动指令

### 小喵统一 Agent 联动前置服务

如果要验证 `stage-web` 聊天、桌面端小喵回复表现或 QQ 普通 AI 回复，需要先启动后端链路：

```powershell
cd F:\xiaomiaoVirtual
conda activate xiaomiao
nanobot serve --config F:\xiaomiaoVirtual\nanobot\.nanobot\config.json
```

然后启动 NapCat，并确认 OneBot WebSocket 是 `127.0.0.1:5004`。

再启动小喵：

```powershell
cd F:\xiaomiaoVirtual\xiaomiao
conda activate xiaomiao
python main.py
```

关键端口：

```text
5519  xiaomiao desktop bridge
8900  nanobot OpenAI-compatible API
```

如果 `python main.py` 输出 bridge 地址后又报 OneBot WebSocket 断开，说明 NapCat 连接失败；主进程退出后 `5519` 也会消失。

### Web 版

```powershell
pnpm dev:web
```

等价于：

```powershell
pnpm -rF @proj-airi/stage-web run dev
```

`stage-web` 的文本输入、移动端输入和页面级录音转文字入口都会请求 `http://127.0.0.1:5519/v1/chat/completions`，再由小喵转发到 `nanobot` 的 `http://127.0.0.1:8900/v1/chat/completions`。

### Web HTTPS 版

```powershell
pnpm dev:web:https
```

### 桌面 Electron 版

```powershell
pnpm dev:tamagotchi
```

等价于：

```powershell
pnpm -rF @proj-airi/stage-tamagotchi run dev
```

桌面端读取小喵 bridge state，将回复同步到字幕、聊天历史、TTS 和 Live2D 口型。

### 文档站

```powershell
pnpm dev:docs
```

### Server Runtime

```powershell
pnpm dev:server
```

### Server API

```powershell
pnpm -F @proj-airi/server dev
```

Server API 会读取 `apps/server/.env.local`。

## 9. 常见问题

### 远端已存在提交，push 失败

如果云端仓库不是空仓库，先确认是否要合并远端历史。

```powershell
git pull origin main --allow-unrelated-histories
git push -u origin main
```

### 修改远端地址

```powershell
git remote set-url origin https://github.com/<你的账号>/<你的仓库>.git
git remote -v
```

### 查看当前 Git 状态

```powershell
git status
git log --oneline --max-count=5
```
